Run the Blender bridge locally

The Generate / sync buttons drive Blender on your own machine, through a small bridge you run locally.

  1. Clone the repo and run npm install.
  2. Open Blender with the BlenderMCP add-on enabled. Its socket port must match the port field in this bar (default 9876).
  3. In the project folder, run npm run bridge — it starts the bridge on http://localhost:7531.
  4. Pick a floor plan in the dropdown → Generate in Blender → edit the walls in Blender (move objects; scale/rotate the root for comfort — it's divided out).
  5. Click ⟳ to pull your edits back and reload the view.
  6. Save the result as .json or .glb.

Blender + the bridge must run on the same machine as your browser. If this site is served over HTTPS, allow the http://localhost connection (Chrome/Edge permit it by default). When the bridge is offline, this preset shows the last committed version.

Make 3D furniture

A 3D piece is built from two optional parts — a solid mesh and its outline — both shown only in the 3D view, scoped to the room. Every furniture symbol carries three editable custom properties: room_ids, 3D, and 3D Outline.

Solid mesh:

  1. Duplicate a furniture symbol (Shift+D) — the copy inherits its room_ids, keeping it linked to that room.
  2. Build it into a solid (extrude / add faces) and fill the faces with the furniture3D material (already in the file).
  3. Set its 3D property to 1.

Outline (optional):

  1. Duplicate a line symbol (or your own line work), keeping its room_ids.
  2. Set its 3D Outline property to 1.

Click ⟳ — the mesh renders flat-white, the outline as dark lines, both shown in 3D only when their room is selected.

Different room? Copy a room_ids value from any asset already in that room and paste it onto yours. In the tools drawer: 3D furniture shows/hides these (on by default); 2D furniture also shows the flat symbols in 3D (off by default). 2D view always shows the flat symbols.

Sun path & bake prep

The ☀ Sun button prepares the open floor plan for an ambient-occlusion / sun-path lightmap bake in one pass. It needs the open .blend to already contain the Inside (AO) material and the Sun Position add-on set to this project's location, date & UTC zone.

  1. Click ☀ Sun. It re-imports the current floor plan, positions the Sun from the project data, and runs interior_bake_prep.
  2. The interior surfaces (walls, caps, floor, furniture, doors) get one shared, non-overlapping Lightmap UV atlas, written back onto the originals — which are otherwise left untouched and gathered in an Original collection, ready to sync back to the viewer.
  3. A copy of those surfaces (plus the ceiling, kept as a hidden-from-camera light occluder) is joined into one Inside_Bake mesh with the Inside material in a Copy collection — this is what you bake.
  4. The timeline is set from sunrise to sunset at 15 min/frame (rounded to the grid, UTC/DST-aware) — scrubbing frames moves the Sun.

Bake the lightmap on the Copy group (into the Inside material's image node), then sync the Original back — the viewer samples the baked texture through the shared Lightmap UVs. Working on the copy keeps the originals' per-asset properties intact.
